Misadventures of Malachi — Design System UI

Audyt pięciu realnych ekranów, katalog odstępstw z decyzjami i wspólny język komponentów UI. Cel: żeby „udawany pixel-art w niskiej rozdzielczości" jak najrzadziej zdradzał, że to udawanie — i żeby te same elementy wyglądały tak samo na każdym ekranie.

Dwie zasady nadrzędne.
  1. Nie zdradzać skalowania. Gra renderuje cały canvas (świat + UI) w logicznej rozdzielczości 1280×720, a potem skaluje go jako jeden obraz. Skalowanie elementów UI = potęga dwójki, minimum liczba parzysta. Wyjątek tylko czcionka. Linie 1px na panelach — do poprawy.
  2. Te same komponenty wszędzie. Jeden sposób na „klawisz", jeden na cień, jedna paleta, jeden minimalny rozmiar czcionki.

Braki do uzupełnienia

do narysowania w Aseprite

4 klawisze strzałek — pixel font nie ma glifu strzałki, dlatego pomoc udaje je trójkątami. Muszą być realnym artem:

  • key_up (↑), key_down (↓)
  • key_left (←), key_right (→)

Rozmiar/styl jak reszta arkusza (16px, skala 2). Najlepiej klawisz-kierunek z trójkątem/strzałką w środku.

można dogenerować kodem

Nie wymagają rysowania — wystarczy rozszerzyć generate_icons:

  • key_, i key_. — dopisać ,. do stringu znaków "<>`[]+-"
  • key_9 — range(0, 9) → range(0, 10) (dziś brak cyfry 9, latentny)

Separatory / i - między klawiszami zostają zwykłym tekstem — to interpunkcja, nie klawisz.

5. Skalowanie i wierność pixel-artu

Bazowa jednostka UI = 2px logiczne. Wszystkie grubości linii, odstępy i rozmiary elementów UI = wielokrotność 2px (albo lepiej 4px). Nine-patch i sprite'y skalowane parzyście (×2, ×4). Skalowanie całego canvasu — najlepiej całkowite krotności.

OdstępstwoGdzieDecyzja
Linie grubości 1px help._draw_cap obwódka width=1; quest._draw_marker width=1 FIX min. 2px (parzyste)
border_radius=4 na chipach/paskach help, quest FIX zaokrąglenie łamie pixel-grid — 0 lub wielokrotność jednostki
UI_BORDER_WIDTH = 9 (nieparzyste) settings.py:398 FIX → 8 lub 10
Ułamkowe skalowanie canvasu DISPLAY_RES_OPTIONS: 1920×1080 → SCALE 1.5; test-env 1024×768 → 0.8 (widać letterboxing i miękkie krawędzie na zrzutach) DOC preferować całkowite krotności (1280×720 → 2560×1440 = 2×)
Nine-patch scale=4, border=6 theme.py:73 OK — wzorzec

6b. Skalowanie ikon emoji / emote

Ikony pixel-art (sentyment na toastach, inline emoji/monety w rich-text) były skalowane ułamkowo do wysokość_fontu × 1.35. Źródła są małe, więc krotność prawie nigdy nie wychodzi całkowita → pygame.transform.scale dubluje część rzędów/kolumn i ikona wygląda na zniekształconą.

Ikona (źródło)CelKrotność (przed)Efekt
emote sentyment 14×1319px (font 14)×1.46 → 20×19 nierówne piksele
moneta / przedmiot 16×1619px (font 14)×1.19 → 19×19 nierówne piksele
Decyzja + FIX (zrobione): tylko całkowita krotność. k = max(1, round(target_h / src_h)), potem scale_by(k). Trzy miejsca: _icon_factor w rich_text.py (inline emoji), toasty w hud.py, emote opcji w dialog.py (_EMOTE_SCALE 1.8 → 2). Toast → ×1 (natywne 16/13, ostre), dialogi (font 16) emote → ×2 (26px). Każdy piksel źródła = czysty blok k×k. Gdy natywny rozmiar za mały → przerysować asset w wyższej rozdzielczości, nie skalować ułamkowo.

7. Panele i pozostałe komponenty

  • Panel standardowy = nine-patch nine_patch_04.png (olive), skala 4, border 6. Questy i pomoc już go używają — wzorzec.
  • Panel dialogu = inny, ciemny nine-patch. świadomy wyjątek — dialog to osobna warstwa (portrety, przyciemnienie świata).
  • Reguły (linie działowe) = kolor #444444, grubość 2px. Spójne między questami a pomocą — do palety jako token RULE.
  • Tag nazwy mówcy (dialog) = nine-patch zakładka, żółty tekst #FFFC67. Udokumentować jako komponent name_tag. (Sprostowanie: „dwa nachodzące tagi" na dlg_b to nie bug — to tabliczka + żółty wikilink [[Barman Absyntnent]] w powitaniu, gdzie NPC przedstawia się imieniem.)
  • Znaczniki stanu (✔/●/○ w questach) = rysowane kształty, nie glify (pixel font ich nie ma). Zachować wzorzec „kształt zamiast tofu".
