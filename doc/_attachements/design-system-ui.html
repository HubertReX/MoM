Dwie zasady nadrzędne

1 · Nie zdradzać, że pixel-art jest skalowany

Gra renderuje cały canvas (świat + UI) w logicznej rozdzielczości 1280×720, po czym skaluje go jako jeden obraz do rozdzielczości ekranu (settings.py SCALE). Dlatego UI skalujemy parzyście, grubości i odstępy to wielokrotności 2px, a border_radius jest zerowy. Jedyny wyjątek dla skalowania ułamkowego to czcionka.

2 · Te same komponenty wszędzie

Jeden sposób na „klawisz" (sprite 32px), jeden na cień (tylko chrome), jedna paleta (tokeny z theme.py), jeden minimalny rozmiar czcionki, jeden wzorzec stopki ze skrótami. Panel po panelu UI rozjeżdżało się — te zasady scalają je w jeden system.

Skalowanie i geometria

  • Bazowa jednostka UI = 2px logiczne. Grubości, odstępy, rozmiary = wielokrotność 2px (lepiej 4px). Żadnych linii 1px.
  • border_radius = 0 (kanty). Świadome wyjątki: pasek sentymentu i scrollbar (schodkowo).
  • Ikony pixel-art skaluj tylko całkowitą krotnością: k = max(1, round(target_h / src_h)), potem scale_by(k).
  • Panel = nine-patch (nine_patch_04.png, scale 4, border 6). Sprite'y i nine-patch skaluj ×2/×4.
  • UI_BORDER_WIDTH = 8 (parzyste), linie działowe RULE 2px.

Komponent „klawisz" (keycap) — zawsze sprite, 32px

Klawisze rysuje wyłącznie ui/keycap.py — nigdy wektorowe chipy ani tekst.

Arkusz HUD.png (16px ×2 → 32px)

Ręczne kafle z ciemnym licem baked (kontrast dla białego glifu):

  • rząd 0: key (pusty), Esc, Tab, Ctl
  • rząd 1: Alt, Enter, Shift, Space
  • rząd 2: strzałki ↑ ↓ → ← (ręczny art)
  • rząd 3: mouse, LMB, RMB

Zasady

  • 32px wszędzie (scale=1.0). Downscale do 16px jest zabroniony.
  • Jednoznakowe — świeży glif na pustym key (generate_icons()).
  • Wieloznakowe / mysz / strzałki — art z arkusza 1:1.
  • Kontrast — ciemne lico baked w HUD.png.

Separatory (to nie klawisze)

  • / — „lub", większy font (FONT_SIZE_LARGE, param sep_font).
  • – — zakres (1–6), krótka szara kreska.
  • pułapka ASCII - to realny klawisz (zoom out).

Cień tekstu — tylko chrome

Model questów: cień tylko na chrome (nagłówki, etykiety, stopki). Proza i glify klawiszy bez cienia. API: _text(..., shadow=False) domyślnie.

Wzorce paneli

Stopka ze skrótami

Skróty klawiszowe idą do stopki (pod linią RULE), nie do nagłówka. Wzorzec: linia + keycap.render_hint. Lewa = zamknięcie/akcje, prawa = hinty kontekstowe (↑ / ↓ przewiń, tylko gdy jest co scrollować).

Scrollbar

Gruby pionowy pasek z zaokrąglonymi końcami, ale schodkowo / kańciasto — helper theme.draw_pixel_round_rect (rogi z pełnych pikseli, jak upscale nearest-neighbour). Nie border_radius= (antyaliasuje). Dodatkowo scroll kółkiem myszy.

Świadome wyjątki od pixel-grid

  • Pasek sentymentu (dialog) — pełny, bez ramki, zaokrąglony po bokach (border_radius = bar_h // 2, pill), czerwony→żółty→zielony wg dyspozycji.
  • Panel dialogu — świadomie inny, ciemny nine-patch (portrety + tabliczka name_tag).

Ekrany (aktualny stan, 1280×720 1:1)

Zrzuty z realnej gry (headless, natywna rozdzielczość canvasu). Każdy klikalny do pełnej rozdzielczości.
