Panel pomocy — mock layoutu (spec współrzędnych)

Cel dokumentu: zatwierdzić układ, grupowanie i treść nowego panelu pomocy zanim powstanie project/ui/panels/help.py. Panel modalny, wyśrodkowany, pauzuje świat do zamknięcia.

Zmiany względem tego mocka w wersji finalnej: pixel font w grze jest znacznie szerszy niż font webowy, więc zamiast 3 kolumn wdrożono 2 kolumny (najdłuższy opis = 378 px). Keycapy są rysowane na żywo jako ciemne chipy z białym tekstem (wysoki kontrast), a nie skalowane sprite'y hotbara. Górny hint skrócono do H / F1 / Esc - zamknij.

Geometria z planu: PANEL_X=90, PANEL_Y=60, PANEL_W=1100, PANEL_H=600 na logicznym 1280×720. Keycapy = te same ciemne przyciski z jasną krawędzią co w grze (icons["key_*"]).

Zachowanie warunkowe: sekcja Debug renderuje się tylko gdy settings.SHOW_DEBUG_INFO jest włączone (` / Z). W trybie web (IS_WEB) sekcja Debug znika w całości, a z sekcji System znikają wiersze F5 / F9 (zapis/odczyt) — tak jak dziś działa [] if IS_WEB. Gdy Debug jest ukryty, trzecia kolumna zawiera tylko „W oknach”, więc panel może być odrobinę niższy albo rozłożyć grupy równiej — do dostrojenia w implementacji.
